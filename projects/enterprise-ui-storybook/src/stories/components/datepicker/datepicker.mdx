# Datepicker

The `DatepickerComponent` is a high-performance, accessible date and time selection
solution engineered for complex enterprise ecosystems. It delivers a unified
interaction model for temporal data entry, combining intuitive graphical
selection with high-efficiency manual input.

## Key Capabilities

- **Intelligent Mode Switching:** Dynamically toggles between date-only and
date-time configurations based on the provided `luxonDateFormat` (or its alias `dateFormat`).
- **Advanced Manual Entry:** Accelerates data input through smart autocomplete,
support for localized German "Uhr" notation, and direct Unix timestamp
processing (seconds and milliseconds).
- **Intelligent Paste Parsing:** Robust clipboard processing that automatically
normalizes separators, handles partial replacements, and supports diverse
formats (ISO, localized, Unix) regardless of the current display format.
- **Flexible Data Binding:** Supports two-way binding via `value` (standard for forms and programmatic updates).
- **Enterprise-Grade Localization:** Seamlessly integrates with the Luxon
library to support diverse international date formats and locales via the `locale` input.
- **Robust Form Integration:** Fully compatible with Angular Reactive and
Template-driven forms via `ControlValueAccessor` and standard validation
protocols.
- **Accessibility by Design:** Adheres to WCAG 2.1 AA standards with comprehensive
keyboard navigation, ARIA live regions, and screen reader optimizations.
- **Precision Time Selection:** Features high-precision time wheels and optional
quick-adjustment controls (`showQuickTimeControls`) for rapid temporal modifications.
- **Testability Hooks:** Supports stable `testId` prefixes for automated testing environments.

## Advanced Manual Entry & Paste Parsing

For experienced developers building high-efficiency data entry interfaces, the `DatepickerComponent` provides a sophisticated manual entry system powered by the `DatepickerPasteParserService`.

### Paste Logic & Normalization

The component implements an "Intelligent Paste" strategy that prioritizes data integrity and user intent:

1.  **Standalone Parsing:** If the clipboard content independently forms a valid date (ISO 8601, localized strings, or Unix timestamps), it is immediately accepted as the new value.
2.  **Separator Normalization:** The parser automatically attempts to map alphanumeric parts of the pasted string to the expected `luxonDateFormat`. This allows users to paste dates with different delimiters (e.g., `2024/07/24` into a `dd.MM.yyyy` field) without manual correction.
3.  **Contextual Merging:** If the pasted text is not a standalone date, it is inserted into the current input at the cursor/selection point. The resulting string is then re-evaluated.
4.  **Unix Epoch Support:** Direct pasting of Unix seconds or milliseconds is supported and automatically converted to the appropriate localized display format.

### Localization & Macros

Manual entry supports Luxon-based localization macros and German-specific notation:
- **"Uhr" Notation:** In date-time mode, the "Uhr" literal is optional during input but preserved in formatting.
- **Localized Months:** Supports full and abbreviated month names based on the active `locale`.
