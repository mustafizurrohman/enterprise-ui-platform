# Testability

The `DatepickerComponent` is designed for seamless integration into automated testing workflows, providing stable hooks for unit, integration, and End-to-End (E2E) testing.

## Automated Testing Hooks

The component supports a `testId` input which serves as a prefix for all internal `data-testid` attributes. This ensures a stable and predictable DOM structure for test selectors, insulating test suites from internal implementation changes.

## Integration Standards

Leveraging standard Angular protocols, the component ensures robust compatibility with modern testing frameworks:
- **Reactive & Template-driven Forms**: Full support for state tracking and validation testing.
- **Async Testing**: Compatible with `fakeAsync` and `waitForAsync` for overlay and animation handling.
- **Signal-based Architecture**: Optimized for modern Angular change detection strategies.

## Testing Paste Functionality

When testing clipboard operations, ensure that:
1.  **Events**: Trigger the `paste` event on the `[data-testid="your-id-input"]` element with a `ClipboardEvent`.
2.  **Feedback**: Verify that `[data-testid="your-id-input-status"]` (the live region) announces the successful insertion (e.g., "Eingefügtes Datum übernommen").
3.  **Validation**: Check that the internal value updates correctly even if pasted separators differ from the configured format.
