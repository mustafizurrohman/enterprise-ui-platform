# Accessibility

The `DatepickerComponent` is engineered for high accessibility, adhering to WCAG 2.1 AA standards to ensure usability for all enterprise users, including those relying on assistive technologies.

## Keyboard Navigation

Full keyboard support is provided for efficient, mouse-free operation:

### Input Focus

| Key | Action |
| :--- | :--- |
| **Enter** / **ArrowDown** | Opens the calendar dialog. |
| **Escape** | Closes the calendar and returns focus to the trigger element. |

### Calendar Navigation

| Key | Action |
| :--- | :--- |
| **Arrow Keys** | Navigates between days in the calendar grid. |
| **Home** / **End** | Moves focus to the first or last day of the current week. |
| **PageUp** / **PageDown** | Switches to the previous or next month. |
| **Shift + PageUp** / **Shift + PageDown** | Switches to the previous or next year. |
| **Space** / **Enter** | Selects the currently focused date. |
| **Escape** | Closes the calendar without making a selection. |

### Time Selection

| Key | Action |
| :--- | :--- |
| **Arrow Keys** | Adjusts the value of the focused time wheel. |
| **Numeric Keys** | Directly enters values for hours, minutes, or seconds. |

## Screen Reader Support

- **ARIA Roles**: Utilizes appropriate ARIA roles including `combobox` for the input, `dialog` for the overlay, `grid` for the calendar grid, and `status`/`alert` for dynamic updates and errors.
- **Live Regions**: Employs multiple `aria-live` regions to announce:
  - **Manual Input**: Feedback during text entry and autocomplete.
  - **Navigation**: Announcements for month and year changes.
  - **Selection**: Confirmation of selected dates and times.
- **Descriptive Labels**: Every date button features a localized, descriptive label (e.g., "Montag, 12. Juli 2026") including information about whether it is today or the currently selected date.
- **Keyboard Shortcuts**: Common actions are mapped to standard keyboard shortcuts and announced via `aria-keyshortcuts` where appropriate.
- **Visual Feedback**: Focus is managed explicitly, ensuring that the active element is always clear to both keyboard and screen reader users.
